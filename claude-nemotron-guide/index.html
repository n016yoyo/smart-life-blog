INTRO

이게 뭔데?

클로드코드 요금이 부담될 때, 터미널 UI·파일 편집·쉘 명령·MCP 연결 같은 사용 경험은 그대로 두고, 뒤에서 실제로 답하는 모델만 NVIDIA가 무료로 풀어놓은 오픈모델(Nemotron 등)로 바꿔치기하는 방법입니다. 오픈소스 래퍼 Nemo Code(by ClawdWorks)가 클로드코드 CLI와 NVIDIA API 사이에 로컬 번역 프록시(LiteLLM)를 세워줘서 가능합니다.

동작 원리: 클로드코드 → (요청) → 로컬 프록시 127.0.0.1:4000 → NVIDIA NIM API → Nemotron 같은 무료 모델이 답 → 다시 클로드코드로. 클로드코드 자체는 안 건드리고, 요청이 가는 주소만 갈아끼우는 방식입니다.
STEP 0 · 복붙

사전 준비물

새 컴퓨터에 Claude Code만 깔린 상태 기준입니다. Node 18+·Python 3.8+가 필요한데, 아래 프롬프트 하나로 확인·설치까지 끝냅니다.

Claude Code에 붙여넣기
node --version, python3 --version을 확인해줘. node는 18 이상, python은 3.8 이상이어야 해. 없거나 낮으면 각각 nvm(node)·pyenv 또는 시스템 패키지 매니저(python)로 설치해줘.
STEP 1 · 직접

NVIDIA 무료 API 키 발급

여기만 브라우저에서 사람이 직접 해야 해요(회원가입이라 터미널로 못 넘김).

1. build.nvidia.com 접속

주소창에 그대로 쳐서 들어가세요. 엔비디아 공식 사이트입니다.

2. 구글/깃허브 계정으로 30초 가입

신용카드 필요 없습니다.

3. 우측 상단 API 키 발급

nvapi-로 시작하는 키가 나옵니다. 이 값을 복사해두세요(설치 단계에서 물어봅니다).

이미 맥북 없어도 최신 AI 다 돌아간다 가이드에서 NVIDIA 키를 발급받았다면 그 키를 그대로 재사용하면 됩니다. 같은 계정, 같은 무료 한도입니다.
STEP 2 · 복붙

Nemo Code 설치

클로드코드 CLI 앞단에 로컬 번역 프록시를 설치하는 오픈소스 스크립트입니다(GitHub: kevdogg102396-afk/free-claude-code, MIT 라이선스). 클로드코드 자체는 다시 설치하지 않고 기존 것을 그대로 씁니다.

Claude Code에 붙여넣기
curl -fsSL https://raw.githubusercontent.com/kevdogg102396-afk/free-claude-code/master/install.sh | bash 를 실행해줘. 설치 중에 NVIDIA API 키를 물어보면 내가 알려줄게. 끝나면 clawdworks 명령이 잡히는지 확인해줘.

Windows는 PowerShell에서 irm https://raw.githubusercontent.com/kevdogg102396-afk/free-claude-code/master/install.ps1 | iex

STEP 3 · 복붙

모델을 Nemotron으로 지정

Nemo Code는 기본으로 Sonnet 슬롯을 다른 모델에 매핑해두는데, NEMO_MODEL 환경변수로 원하는 NVIDIA 모델을 직접 지정할 수 있습니다.

Claude Code에 붙여넣기
NEMO_MODEL 환경변수를 nvidia/nemotron-3-super-120b-a12b 로, NVIDIA_API_KEY 환경변수를 내가 알려준 값으로 설정해줘. 그다음 clawdworks run "간단한 파이썬 함수 하나 짜줘" 로 실제로 잘 도는지 테스트해줘.
슬롯기본 매핑Nemotron으로 바꾸려면
Sonnet(기본)moonshotai/kimi-k2.6NEMO_MODEL=nvidia/nemotron-3-super-120b-a12b
Opusqwen/qwen3.5-397b-a17b고정(대형 추론용)
Haikuminimaxai/minimax-m2.7고정(빠른 작업용)

모델 ID는 NVIDIA 쪽에서 수시로 바뀔 수 있어요. 안 잡히면 clawdworks models로 현재 목록을 다시 확인하세요.

STEP 3.5 · 골라 부르기

터미널에서 claude-nemotron / claude-glm / claude-minimax 로 골라 부르기

릴스에서 본 것처럼, 터미널에 명령어 한 단어로 원하는 모델을 부를 수 있어요. 설치는 STEP 2에서 이미 한 번 끝났고(모델마다 다시 깔 필요 없음), 여기선 그 위에 명령어 3개(alias)만 얹습니다. 모델을 바꾸는 스위치는 NEMO_MODEL 하나라, 각 모델을 미리 묶어두면 됩니다.

명령어부르는 모델언제 쓰나실측 상태
claude-nemotronnvidia/nemotron-3-ultra-550b-a55b코딩·범용 (기본)✓ 잘 됨 4~16초, 가끔 순간 혼잡(재시도)
claude-glmz-ai/glm-5.2한국어·다국어 긴 답△ 느림 20초대에 되기도, 혼잡하면 실패도
claude-minimaxminimaxai/minimax-m2.7길고 복잡한 추론△ 생각형 사고를 많이 함, 짧은 답엔 과함
이 3개 명령어 만들기(alias): 아래를 ~/.zshrc(맥) 또는 ~/.bashrc(리눅스)에 붙여넣고 저장하세요. 새 터미널을 열면 세 명령어가 바로 잡힙니다.
~/.zshrc 에 추가
alias claude-nemotron='NEMO_MODEL=nvidia/nemotron-3-ultra-550b-a55b clawdworks'
alias claude-glm='NEMO_MODEL=z-ai/glm-5.2 clawdworks'
alias claude-minimax='NEMO_MODEL=minimaxai/minimax-m2.7 clawdworks'

이제 터미널에서 명령어만 바꿔 부르면 그 모델이 답합니다. 예:

이렇게 치면일어나는 일
claude-nemotron run "파이썬 함수 짜줘"Nemotron이 코드로 답
claude-glm run "이 문단 자연스러운 한국어로 다듬어줘"GLM이 한국어로 답(조금 느림)
claude-minimax run "이 로직 단계별로 길게 설명해줘"MiniMax가 길게 추론해서 답
셋 다 같은 NVIDIA 키·같은 무료 한도를 씁니다. 무료 API라 GLM·MiniMax는 혼잡하면 느리거나 응답이 안 올 수 있으니, 일상 코딩은 가장 안정적인 claude-nemotron을 기본으로 두고 필요할 때만 나머지를 부르세요. 모델 ID가 안 잡히면 clawdworks models로 현재 목록을 확인해 alias의 ID를 맞춰주세요.
EVIDENCE

Nemotron이 실제로 이렇게 답합니다

클로드코드용 프록시(clawdworks)를 통한 화면은 아니지만, 같은 NVIDIA Nemotron 모델을 저희가 이미 운영 중인 무료 API 라이브 데모 백엔드로 방금 직접 호출해서 받은 진짜 응답입니다(조작·캡처 없음).

LIVE · nvidia/nemotron-3-ultra-550b-a55b 실제 호출
Q. 클로드코드 터미널에서 무료 Nemotron 모델을 쓰는 이유를 한 문장으로 설명해줘
A. 로컬에서 무료로 고성능 코딩 모델을 실행해 API 비용 없이 프라이빗하게 개발할 수 있어서입니다.

동일 NVIDIA 키·동일 모델을 쓰는 프록시라 Nemo Code로 붙여도 같은 품질의 답을 받습니다.

CAUTION

주의할 점

⚠ 무료 모델은 Claude가 아님 ⚠ 프롬프트 인젝션에 더 취약 ⚠ Anthropic 비공식·비제휴
Nemo Code 공식 안내: 오픈모델은 강력하지만 악성 파일·웹페이지·붙여넣은 콘텐츠로 인한 프롬프트 인젝션에 Claude보다 취약할 수 있습니다. 개인 PC에선 Docker 모드(격리) 권장, 신뢰 못 하는 콘텐츠는 전체 접근 권한 에이전트에 물리지 말 것. Nemo Code는 클로드코드 CLI를 번들·재배포하지 않고 npm 런타임 의존성으로 설치하며, Anthropic과 제휴 관계가 아닙니다.
