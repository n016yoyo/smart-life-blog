블랙홀을 만들라고 시켰습니다

클로드 페이블5 vs 옵퍼스 4.8 · 같은 주문 · 딱 한 번
주문 한 줄 수정 기회 0번 실패 화면도 그대로 공개

대결 규칙

  • 두 모델에게 완전히 같은 주문을 한 번만 보냈어요. 파일 하나를 원샷으로 작성, 테스트·수정 금지.
  • 결과물은 손대지 않았습니다. 아래 카드를 탭하면 그 원본이 그 자리에서 실시간 렌더링돼요.
  • 블랙홀 물리는 진짜입니다. 아인슈타인 일반상대성이론 방정식을 코드로 직접 풀게 했어요.
ROUND 1 · 블랙홀

그냥 블랙홀은 둘 다 만들었어요

인정할 건 인정해야죠. 이 난이도에선 격차가 없었습니다.

FABLE 5PASS▶ 탭하면 실시간 렌더 새 탭에서 크게 (폰 OK)
OPUS 4.8PASS▶ 탭하면 실시간 렌더 새 탭에서 크게
판정: 무승부. 페이블 쪽이 색감·포톤 링이 더 풍부하긴 하지만, 둘 다 "휘어진 빛"을 제대로 계산했습니다.
ROUND 2 · 회전 블랙홀

난이도를 올리자 격차가 벌어졌습니다

회전하는 커(Kerr) 블랙홀. 물리 난이도가 한 단계 위입니다.

FABLE 5PASS▶ 탭하면 실시간 렌더 새 탭에서 크게 (폰 OK)
OPUS 4.8FAIL · 0fps▶ 탭해서 검은 화면 직접 확인 새 탭에서 크게
판정: 페이블5 승. 옵퍼스는 30초를 기다려도 검은 화면에 0fps였습니다(위 링크에서 직접 확인 가능). 페이블5는 비대칭 그림자까지 그려냈어요.

위 결과물 4개 전부, 우리가 한 글자도 안 고친 원샷 원본입니다.

둘에게 보낸 주문 전문

이걸 그대로 복붙하면 여러분 클로드도 블랙홀을 만들어 줍니다.

ROUND 1 프롬프트 (블랙홀) 
웹브라우저에서 실시간으로 돌아가는 블랙홀 렌더러를 HTML 파일 하나로 만들어줘. 조건:
1) 일반상대성이론 슈바르츠실트 시공간에서 광자 측지선을 실제 수치적분으로 계산하는 레이트레이싱(가짜 렌즈 왜곡 필터 금지)
2) 인터스텔라처럼 얇은 강착원반이 블랙홀 위아래로 휘어 감겨 보일 것, 포톤 링 표현
3) 배경 별들이 중력렌즈로 왜곡될 것
4) 도플러 빔잉으로 원반 한쪽이 더 밝을 것
5) WebGL 셰이더 실시간 렌더링, 외부 라이브러리·외부 리소스 금지
6) 카메라는 천천히 궤도를 돌 것, 9:16 세로(1080x1920) 화면 기준
ROUND 2 프롬프트 (회전 블랙홀) 
회전하는 커(Kerr) 블랙홀 렌더러를 HTML 파일 하나로 만들어줘. 조건:
1) 커 시공간(스핀 a=0.9M)의 광자 측지선을 실제 수치적분으로 풀 것(보이어-린드퀴스트 좌표의 해밀토니안 방정식 또는 동등 수준). 슈바르츠실트에 소용돌이 텍스처 입히는 가짜 금지
2) 프레임 드래깅이 실제로 보일 것: 그림자가 비대칭(D자형)이고 포톤링이 한쪽으로 쏠릴 것
3) 강착원반(ISCO 안쪽 비움)·도플러 빔잉·배경별 중력렌즈 포함
4) WebGL 셰이더 실시간, 외부 리소스 금지, 9:16 세로 1080x1920 기준, 카메라 느린 궤도

⚠️ 정직하게: 페이블도 완벽하진 않았어요

라운드2 페이블 결과물에는 세로줄 흠집(극축 좌표 특이점)이 남아 있습니다. 원샷 기준이라 그대로 공개해요. "한 번 더 고쳐줘"라고 하면 지워지지만, 그건 원샷이 아니니까요.

따라 하는 법 (1분)

  1. 클로드 코드(Claude Code)를 켜요. 설치가 처음이면 비개발자 완전 정복 가이드 참고.
  2. 위 전체 복사를 눌러 프롬프트를 그대로 붙여넣어요.
  3. 클로드가 만들어 준 HTML 파일을 더블클릭하면 끝. 브라우저에서 블랙홀이 돌아갑니다.
  4. 모델 선택이 되면 페이블5로. 라운드2(회전 블랙홀)는 다른 모델로는 이 페이지처럼 실패할 수 있어요.
