텔레그램으로 Claude Code 조종하기
만드는 법 완전 가이드

폰 텔레그램에서 메시지를 보내면, 하나의 살아있는 Claude Code 세션이 실제 작업(파일 수정, ffmpeg, 스크랩)을 하고 결과를 다시 톡으로 답장합니다. 추가 서버 0개, 봇 토큰 1개면 끝.

뭘 만드나 전체 구조 동작 원리 ⚠️ 함정·노하우 🤖 AI 빌드 프롬프트 🙋 사람이 할 일 비용 FAQ

🎯 뭘 만드는 거야?

나(폰): "릴스 자막 좀 키워줘" 💬 ↓ (봇이 톡을 Claude Code 로 전달) Claude Code: ffmpeg 로 자막 키우고 파일 저장 ↓ 봇: "✅ 완료. out/reel_final.mp4 만들었어요" 📩

PC 앞에 없어도 폰 톡 한 줄로 진짜 작업이 돌아갑니다. 대화가 한 세션으로 계속 이어져서, 어제 하던 작업을 오늘 이어서 시킬 수도 있어요. 웹훅도, 클라우드 서버도 필요 없습니다.

🗺️ 전체 구조

조각은 단 3개뿐입니다. 별도 인프라가 없는 게 핵심이에요.

① 텔레그램 봇 📱
@BotFather 로 1분 발급. 메시지 받는 창구. 토큰 1개가 전부.
② 브리지 (bridge.js) 🌉
내 PC의 Claude Code 환경에서 도는 작은 Node 스크립트. 의존성 0(내장 모듈만).
③ Claude Code 세션 🤖
브리지가 "스트리밍 모드로 하나만" 띄워두고 살려두는 실작업 두뇌.
＋ 상태 파일 🗄️
세션 id 와 메시지 offset 저장. 재시작해도 대화 이어감.
폰 텔레그램 ──(메시지/파일)──▶ Telegram 서버 (약 24h 보관) ▲ │ getUpdates (long-poll) │ ▼ [ bridge.js ] (Claude Code 환경에서 실행) │ ※ claude 를 스트리밍 모드로 하나만 유지 │ stdin ◀── 텔레그램 메시지 (JSON 한 줄) │ stdout ──▶ 결과 이벤트 ▼ 같은 세션 = 캐시 따뜻 = 저렴 → 답장 (sendMessage)
💡 웹훅·서버가 필요 없는 이유: 텔레그램의 getUpdates long-poll 로 브리지가 직접 "새 메시지 있어?"를 물어봅니다. 그래서 공개 주소도, 클라우드 배포도 없이 내 PC만으로 돕니다.

⚙️ 동작 원리 검증된 스트리밍 프로토콜

핵심: 프로세스 하나를 계속 살려둔다

메시지마다 claude -p --resume 를 새로 부르면 매번 대화를 처음부터 다시 읽어 캐시가 차갑고 비쌉니다. 대신 프로세스 하나를 계속 살려두면 대화가 한 세션으로 쌓이고 프롬프트 캐시가 따뜻하게 유지돼서, 터미널에서 직접 치는 것과 같은 저렴한 비용이 됩니다.

claude -p --input-format stream-json --output-format stream-json --verbose [--resume <id>]

주고받는 형식

방향형식
입력 (stdin, 메시지당 한 줄){"type":"user","message":{"role":"user","content":[{"type":"text","text":"..."}]}}
시작 이벤트{"type":"system","subtype":"init","session_id":"..."} (1회)
답 조각{"type":"assistant","message":{...}}
턴 종료{"type":"result","result":"답","session_id":"..."}

브리지는 result 이벤트가 올 때까지 기다렸다가 .result(답)를 톡으로 보내고, .session_id 를 저장합니다(재시작 시 --resume 으로 부활).

✅ --verbose 는 stream-json 출력에 필수입니다. 실행 시 apiKeySource:none 이 보이면 구독으로 동작(별도 유료 API 키 불필요)한다는 뜻이에요.

⚠️ 우리가 겪은 함정 & 노하우 이게 핵심

이 가이드의 진짜 가치. 직접 만들며 부딪힌 부분들입니다.

💸 비용은 "캐시"가 좌우한다

같은 세션을 빠르게 주고받으면(캐시 TTL 약 5분 안) 큰 컨텍스트도 캐시 읽기(약 0.1배)로 싸집니다. 5분 넘게 비면 캐시가 식어 그 한 메시지는 비싸져요. 이 브리지가 지속 세션을 쓰는 이유가 바로 이 따뜻한 캐시를 재현하려는 것입니다.

🐘 거대한 기존 대화를 봇으로 매번 이어가지 말 것

수십 MB짜리 큰 대화를 봇으로 resume 하면 메시지당 컨텍스트 토큰이 폭증합니다. 봇 전용은 가볍거나 새 세션으로 쓰고, 필요하면 MODEL=claude-sonnet-4-6 으로 비용을 낮추세요.

🔌 spawn ENOENT: claude 경로 문제

백그라운드로 띄우면 spawn claude ENOENT 가 날 수 있어요. PATH 에 ~ 가 안 풀려 claude 를 못 찾는 경우입니다. 실행 스크립트에서 claude 절대경로를 잡아주면(예: CLAUDE_BIN="$(command -v claude)") PATH 상태와 무관하게 안정적으로 켜집니다.

🔁 터미널과 봇에서 같은 세션 동시 구동 금지

같은 세션을 터미널과 봇에서 동시에 돌리면 대화 기록(transcript)이 갈라집니다. 한 번에 한 곳에서만 쓰세요.

🆔 세션 id 안 외워도 됨 (번호 피커)

/sessions 로 최근 대화 목록(번호·시각·미리보기)을 보고 /resume <번호> 로 이어갑니다. 터미널 claude --resume 피커처럼 긴 id 를 외울 필요가 없어요. 목록은 ~/.claude/projects/<workdir 인코딩>/*.jsonl 을 최근순으로 읽어 첫 메시지를 미리보기로 보여줍니다.

💀 환경이 죽어도 메시지는 안 날아간다

텔레그램이 메시지를 약 24시간 보관하므로, 브리지가 그 안에 다시 켜지면 유실 없이 처리됩니다. 더 길게 보장하려면 텔레그램 → Cloudflare Worker 웹훅 → KV 큐로 확장하면 영구 보관이 됩니다.

🤖 AI 빌드 프롬프트 복사해서 AI에 붙여넣기

아래를 통째로 복사해 Claude Code 등 AI 코딩 도구에 붙이면, 브리지 스크립트 전체를 만들어줍니다. (봇 토큰 발급은 아래 사람 체크리스트 참고)

텔레그램으로 Claude Code 를 조종하는 브리지(bridge.js)를 만들어줘. Node 내장 모듈만 쓰고(외부 의존성 0), 어떤 Claude Code 프로젝트에도 복붙으로 작동하게 해.

# 동작 개요
- 텔레그램 getUpdates long-poll 로 메시지를 받는다(웹훅·서버 불필요).
- claude 를 스트리밍 모드로 "하나만" 띄워 계속 살려둔다(따뜻한 캐시 = 저렴).
- 받은 텍스트를 그 세션 stdin 에 JSON 한 줄로 넣고, result 이벤트의 답을 톡으로 보낸다.

# Claude 프로세스
spawn 명령: claude -p --input-format stream-json --output-format stream-json --verbose --dangerously-skip-permissions [--resume <id>]
- claude 실행 파일은 process.env.CLAUDE_BIN || "claude" 로(절대경로 주입 가능, ENOENT 방지).
- stdin 입력(한 줄): {"type":"user","message":{"role":"user","content":[{"type":"text","text":"<메시지>"}]}}
- stdout 파싱: type=="system"&&subtype=="init" 또는 result 이벤트의 session_id 를 저장,
  type=="result" 의 result 를 답으로 사용. 프로세스 죽으면 자동 재시작 후 --resume.

# 텔레그램
- getUpdates(offset, timeout=50) long-poll 루프. 처리한 update_id+1 을 offset 으로 저장.
- sendMessage 로 답장(긴 답은 4096자 단위로 분할).
- 문서/사진 첨부: getFile→다운로드해 WORKDIR 에 저장(중복 시 name(1).ext), 캡션 있으면 지시문으로 전달.

# 명령어
/help,/start(안내+chat_id) · /sessions,/ls(최근대화 목록: 번호·시각·크기·미리보기, 현재세션 표시)
/resume <번호|id> · /session(현재 id+터미널 resume 명령) · /new,/clear(새 세션) · /cd <경로>(작업폴더 변경)
/down <경로>(서버 파일을 톡으로 전송). 그 외 텍스트는 Claude 로 전달.

# 환경변수
TG_TOKEN(필수) · ALLOWED_CHAT(이 chat_id 만 허용, 보안 필수) · WORKDIR(기본 cwd)
RESUME(이어갈 세션 id) · MODEL(예 claude-sonnet-4-6) · STATE_FILE(기본 ./.tg_bridge_state.json) · CLAUDE_BIN

# 반드시 반영할 함정
- 지속 세션을 유지해 캐시를 따뜻하게(메시지마다 새로 resume 하지 말 것).
- claude 절대경로(CLAUDE_BIN)로 spawn ENOENT 방지.
- ALLOWED_CHAT 없으면 사실상 셸 권한 노출이므로 본인 chat 만 허용.
- 세션 목록은 ~/.claude/projects//*.jsonl 을 최근 수정순으로 읽어 첫 메시지를 미리보기로.
- 파일은 텔레그램 한도 20MB.

bridge.js 전체 코드 + 한 방 실행/재시작용 restart.sh(토큰·ALLOWED_CHAT·CLAUDE_BIN export, nohup 백그라운드, stop 인자) 까지 만들어줘.

붙여넣으면 AI가 코드를 만들고, "봇 토큰부터 발급하세요"라고 안내해줄 거예요.

🙋 사람이 직접 해야 하는 것 AI가 대신 못 함

봇 발급과 실행은 본인이 해야 합니다. 순서대로:

1
텔레그램 봇 생성
@BotFather 에게 /newbot → 이름 정하고 토큰 발급(123456:ABC-...)
2
내 chat_id 확인
TG_TOKEN="..." node bridge.js 실행 후, 봇에게 /start → "chat_id=99999999" 회신
3
본 실행 (본인 chat만 허용)
TG_TOKEN="..." ALLOWED_CHAT="99999999" WORKDIR="$(pwd)" node bridge.js
4
백그라운드로 유지
nohup node bridge.js > bridge.log 2>&1 & (터미널 닫아도 유지). 끄기는 pkill -f "node bridge.js"
5
이제 폰에서 톡
보내는 메시지가 그 프로젝트의 Claude Code 작업으로 실행되고 결과가 답장됩니다

상황비용
가벼운 새 세션, 빠른 왕복저렴 (따뜻한 캐시 = 약 0.1배)
5분 넘게 비운 뒤 한 메시지그 한 번은 캐시가 식어 비쌈
수십 MB 큰 대화를 매번 resume메시지당 토큰 폭증 (피할 것)

참고: 컨텍스트 로딩 때문에 사소한 "PONG" 한 번도 약 $0.06 상당 quota 를 씁니다. 큰 대화 resume 은 그 수배에서 수십배. 절약하려면 봇 전용은 가벼운 세션 + 필요 시 MODEL=claude-sonnet-4-6.

❓ FAQ

Q. 서버나 클라우드가 필요해?
❌ 아니요. 내 PC의 Claude Code 환경에서 node bridge.js 하나만 돌면 됩니다. 텔레그램 long-poll 이라 공개 주소가 필요 없어요.
Q. 돈이 따로 들어?
구독으로 도는 경우 현금 청구는 0원, 구독 quota 만 소모합니다. 캐시를 따뜻하게 쓰면 인터랙티브와 같은 저렴한 비용이에요.
Q. PC를 꺼두면 메시지가 날아가?
텔레그램이 약 24시간 보관해서, 그 안에 브리지를 다시 켜면 유실 없이 처리됩니다.
Q. 보안은?
--dangerously-skip-permissions 로 도구가 자동 실행되니, 봇을 아는 사람은 사실상 셸 권한을 가집니다. 반드시 ALLOWED_CHAT 으로 본인 chat 만 허용하고, 토큰 노출 시 @BotFather /revoke.
Q. 기성품은 없어?
같은 일을 하는 완성품 cokacdir 가 있습니다(텔레그램·디스코드·슬랙 멀티플랫폼). "기능 풍부한 기성품"이면 그걸, "직접 통제·최소 구성"이면 이 키트를 쓰면 됩니다.
