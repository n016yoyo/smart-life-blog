INTRO

이게 뭔데?

블로그를 매일 쓰는 건 진짜 고됩니다. 그래서 Claude Code에게 브라우저 조종을 맡겼습니다. Claude가 짠 코드가 크롬을 열고, 실제 네이버 스마트에디터에 들어가서 제목과 본문을 한 글자씩(사람 속도로) 타이핑하고, 준비한 이미지를 문단 사이에 자동으로 끼워 넣고, 임시저장까지 합니다.

동작 원리: Claude Code → (자동화 스크립트) → 브라우저 조종 도구 Playwright → 진짜 크롬 창 → 네이버 에디터에서 키보드·마우스를 사람처럼 조작. 네이버 입장에선 "누가 손으로 쓰는 것"과 구분이 안 됩니다.
READ FIRST

먼저, 정직하게 짚고 갑니다

자동화는 임시저장까지입니다. 발행 버튼은 사람이 누르세요. 두 가지 이유예요.

  • 사고 방지 — 검토 없이 자동 발행하면 오타·잘못된 사진·이상한 문장이 그대로 공개됩니다. 초안은 AI, 최종 확인은 사람 — 이게 안전합니다.
  • 계정 보호 — 대량 자동 발행은 네이버가 스팸으로 볼 수 있습니다. 하루 1~2건, 검토 후 발행이 안전한 습관입니다.
네이버 공식 "글쓰기 API"는 없습니다. (2026년 현재 오픈 API는 검색만 제공) 그래서 이 방법은 실제 에디터를 자동으로 조작하는 방식이고, 그만큼 부계정으로 시작하기를 권합니다.
STEP 0 · 복붙

사전 준비물

새 컴퓨터에 Claude Code만 깔린 상태 기준입니다. 나머지는 아래 프롬프트 하나로 Claude가 알아서 설치합니다.

Claude Code에 붙여넣기
네이버 블로그 자동 글쓰기를 만들 거야. node 18 이상이 있는지 확인하고, 없으면 설치해줘. 그리고 브라우저 자동화용 playwright-core를 설치해줘. 크로미움은 시스템에 있는 걸 쓸 거야.
✓ Claude Code ✓ Node 18+ ✓ 크롬(또는 크로미움) 네이버 부계정
STEP 1 · 5분

네이버 로그인, 한 번만 넘겨주기

자동화가 "로그인된 나"로 동작하려면 로그인 정보(쿠키)를 한 번 넘겨주면 됩니다. 매번 할 필요 없이 처음 한 번뿐이에요.

  1. 크롬에서 naver.com 접속 → 로그인 ("로그인 상태 유지" 체크)
  2. F12 → 상단 Application 탭 → 왼쪽 Cookies → https://www.naver.com
  3. 필터 칸에 NID 입력 → NID_AUT, NID_SES 두 값 복사
  4. Claude Code에게 그 두 값을 주고 "쿠키 저장해줘"라고 하면, 안전한 곳(.gitignore 경로)에 보관합니다.
⚠ 함정 1 · NID가 안 보이면

필터에 NID 쳤는데 아무것도 없다면, 그 창이 로그인 안 된 상태입니다. NID_AUT·NID_SES는 로그인하는 순간 생기는 쿠키라, 로그인부터 하고 다시 확인하세요.

NID_AUT·NID_SES는 사실상 아이디+비번급입니다. 공개 채팅방·저장소에 올리지 말고, 부계정으로 쓰세요.
STEP 2 · 복붙

글감 주고, 실행하기

어떤 글을 쓸지 Claude에게 알려주고, 한 줄로 실행합니다. 글 내용·사진은 Claude가 생성하게 하거나, 직접 준비한 걸 넣어도 됩니다.

Claude Code에 붙여넣기
이 주제로 네이버 블로그 초안을 써줘: (여기에 주제).
그리고 이 자동화 규칙 세 가지로 실행해줘.
하나, 브라우저를 열어 실제 네이버 스마트에디터에 들어가 줘.
둘, 제목과 본문을 사람 속도로(한 글자씩) 타이핑하고, 문단 사이에 사진을 자동으로 넣어 줘.
셋, 발행은 하지 말고 임시저장까지만 해줘. 전 과정은 영상으로 녹화해 줘.
결과: 크롬이 뜨고 → 제목이 또박또박 쳐지고 → 본문이 촤르륵 → 사진이 척척 들어가고 → 임시저장(저장 카운트가 올라감). 끝나면 out/ 폴더에 전 과정 녹화 영상이 저장됩니다.

발행은 네이버 블로그 앱이나 웹에서 초안을 열어 검토한 뒤 발행 버튼만 누르면 됩니다.

우리가 먼저 밟은 지뢰

실전 함정 6가지

직접 만들면서 실제로 막혔던 지점들입니다. Claude에게 "이 함정 조심해서 해줘"라고 같이 주면 시행착오가 줄어요.

1 · "취소"가 "취소선"을 누른다

팝업 닫기 버튼을 "취소" 포함으로 찾으면, 에디터 툴바의 "취소선" 버튼을 눌러서 본문이 전부 취소선으로 써집니다. 정확히 일치(text-is)로 찾게 하세요.

2 · 글쓰기 진입 URL은 딱 하나만 통과

구형 진입점(GoBlogWrite.naver 등)은 재로그인으로 튕깁니다. blog.naver.com/{내아이디}/postwrite 이 신형 주소만 바로 열립니다.

3 · "작성 중인 글 복구" 팝업

에디터 열면 이전 임시저장 복구 팝업이 뜹니다. 팝업 안에서만 "취소"(=새로 쓰기)를 누르게 하세요(함정 1과 세트).

4 · 쿠키 하나라도 형식 틀리면 전체 실패

쿠키에 도메인 값이 빠지면 "domain/path pair" 에러로 전부 안 들어갑니다. 하나씩 넣어 보며 범인을 찾게 하세요.

5 · 녹화하려면 ffmpeg 연결

Playwright 화면 녹화는 자체 ffmpeg를 요구합니다. 시스템 ffmpeg를 심볼릭 링크로 걸어주면 됩니다(Claude가 처리).

6 · 자동화 후엔 녹화를 프레임 단위로 확인

스크린샷 한 장으론 "언제부터 꼬였는지" 모릅니다. 녹화를 몇 초 간격으로 잘라 눈으로 확인하면 문제 지점이 바로 보입니다.
