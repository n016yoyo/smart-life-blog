# 작업 규칙

## 시작 전 항상
- about-me.md 와 voice.md 를 먼저 읽어라.
- 바로 실행하지 말고, 불명확하면 먼저 질문해라.

## 결과물
- 문서·콘텐츠는 한국어로. 웹은 HTML 한 파일로.
- 항상 내가 바로 쓸 수 있는 완성형으로.

## 하지 말 것
- 내 허락 없이 파일 삭제 금지.
- "할 수 있습니다" 같은 늘어지는 AI 말투 금지.
ABOUT ME

about-me는 "질문 형식"으로 만들어라

직접 빈칸 채우기 어렵지? 클로드한테 인터뷰를 시켜. 한 번에 하나씩 물어보고, 답하면 알아서 about-me.md랑 voice.md로 정리해줘.

인터뷰 프롬프트 (앱에 그대로 붙여넣기) 
나에 대한 about-me.md 와 voice.md 를 만들고 싶어.
네가 인터뷰어가 되어서 아래 질문을 한 번에 하나씩만 물어봐.
내가 답하면 다음 질문으로 넘어가. 7개 다 끝나면 요약 보여주고,
확인받은 다음에 about-me.md 와 voice.md 두 파일로 저장해.

질문:
1. 무슨 일을 해? (직업 / 전문 분야 한 줄)
2. 주요 고객이나 타깃 독자는 누구야?
3. 주로 어떤 결과물을 만들어? (콘텐츠 / 웹 / 자동화 등)
4. 어떤 말투를 원해? (격식체 / 친근체 / 직접적 / cold·confident 등)
5. 절대 쓰기 싫은 표현·단어가 있어? (예: "정말", "너무", "함께")
6. 네가 잘 썼다고 생각하는 글·작업 하나 통째로 붙여줄게. (붙여넣기)
7. 클로드가 항상 지켜야 할 규칙 1~2개는?
6번이 제일 강력해. 지시 백 줄보다 잘 된 글 원문 하나가 내 톤을 더 잘 가르쳐.

MAKE YOUR SKILL

첫 스킬 직접 만들기

SKILL.md는 두 부분: 맨 위 frontmatter (name, description) + 그 아래 본문 (지시사항).

description이 제일 중요해. 클로드가 이 문장을 보고 "이 작업에 이 스킬 켤까?"를 판단해. 내가 실제로 칠 법한 말 + 동작 동사로 써.

방법 A. 클로드가 알아서 만들기

방금 한 작업 → 스킬 자동 생성
방금 한 캐러셀 작업을 재사용할 수 있게 스킬로 만들어줘.
~/.claude/skills/carousel/SKILL.md 로 저장하고,
내가 "캐러셀 만들어줘"라고 하면 자동으로 켜지게 description 써줘.

방법 B. 내 규칙 직접 넣기

캐러셀 스킬 예시 (붙여넣기용)
아래 내용으로 ~/.claude/skills/carousel/SKILL.md 스킬 파일 만들어줘:

---
name: carousel
description: 인스타 카드뉴스 / 캐러셀 만들 때 사용. "캐러셀 만들어줘", "카드뉴스로 정리해줘" 요청 시 발동. 9 슬라이드 구조로 출력.

# 캐러셀 생성 규칙

## 구조 (9 슬라이드)
- 1번: 커버 = 큰 후크 제목 (숫자 / 부정 / 역설), 캐릭터 없음
- 2~8번: 본문 = 슬라이드당 핵심 1개. 30% 이상에 비주얼
- 9번: CTA

## 글쓰기
- voice.md 톤 따르기. "정말/너무/엄청" 금지.
- 첫 줄은 무조건 후크.

## 출력
- 슬라이드별 제목 + 본문 + 비주얼 지시를 표로.
새 세션 열고 /carousel 치거나 "캐러셀 만들어줘" 해봐. 안 켜지면 description에 내가 쓸 법한 말 더 추가.

HOOKS

훅(Hooks) 시스템 자동화

클로드 코드는 세션 시작·종료·작업 전후에 자동으로 뭔가를 실행하게 설정할 수 있어.

🌅
SessionStart (세션 시작 시)

예: 세션 열릴 때마다 파일명 자동 정리, 환경 세팅 스크립트 실행

🌙
SessionEnd (세션 종료 시)

예: 작업 결과 자동 요약, 로그 저장

⚡
PreToolUse / PostToolUse

예: 특정 도구 실행 전·후에 검증 스크립트 자동 실행

설정 방법

~/.claude/settings.json에 hooks 항목 추가.

settings.json 예시
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/my-scripts/session-start.sh"
          }
        ]
      }
    ]
  }
}
훅은 클로드가 아니라 컴퓨터가 직접 실행해. 반복되는 세팅 작업을 완전 자동화할 수 있어.
MEMORY SYSTEM

메모리 시스템 레벨업 장기 운영

기본 memory.md 하나에서 타입별로 분리하면 훨씬 정교하게 기억해.

TYPE 1
user

내 역할·목표·전문분야. "나는 어떤 사람인지"를 기억.

TYPE 2
feedback

"이렇게 하지 마" / "이거 잘 됐어" 피드백. 실수 방지 핵심.

TYPE 3
project

진행 중인 작업·결정사항·마감일. 매일 업데이트.

TYPE 4

멀티 세션 운영

클로드 코드는 여러 폴더를 동시에 작업할 수 있어. 탭 여러 개처럼.

✅ 가능한 것

폴더 A에서 코딩하면서 폴더 B에서 문서 작업. 완전 별도로 돌아가.

⚠️ 주의할 것

두 세션이 같은 파일을 동시에 수정하면 충돌 발생. 파일은 한 세션이 담당하게 분리.

사용량 한도는 세션 수와 상관없이 계정 전체가 공유해. 여러 세션을 동시에 돌리면 한도가 더 빨리 소진돼.
